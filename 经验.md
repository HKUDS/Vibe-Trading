# Vibe-Trading 部署经验总结

## 问题描述

部署 Vibe-Trading 到 ECS200 后，聊天功能报错：
```
Execution failed: 404 page not found
```

## 排查过程

### 1. 确认 .env 配置正确

检查 `/home/anders/projects/vibe-trading/agent/.env`：
```
LANGCHAIN_PROVIDER=minimax
MINIMAX_API_KEY=sk-F7HNrgA2Iaa3F0cxr4o7zmx6pq5yIo4cMDyxojwzJ8bcb
MINIMAX_BASE_URL=https://maas.gd.chinamobile.com:36007/ai/uifm/open/v1
LANGCHAIN_MODEL_NAME=minimax-m25
API_AUTH_KEY=vibe-trading-cb178040011780b55520817b96360389
```

### 2. 本地 curl 测试 API 正常

curl 测试 MiniMax API 端点返回正确的响应，证明 API key 和 URL 配置本身没问题。

### 3. 发现问题根源

检查进程环境变量：
```bash
cat /proc/<pid>/environ | tr '\0' '\n' | grep MINIMAX
```

发现进程中的 `MINIMAX_BASE_URL` 是 `https://api.minimaxi.com/anthropic`（错误值），而不是 `.env` 中的正确值。

进一步检查 `~/.bashrc`，发现：
```bash
# MiniMax API Configuration (added 2026-04-10)
export MINIMAX_API_KEY="sk-cp-pp_OkGW4o-KMGjBGsou..."
export MINIMAX_BASE_URL="https://api.minimaxi.com/anthropic"
```

### 4. 根因

启动命令 `vibe-trading serve` 会 spawn 一个 bash subshell，该 subshell 会 source `~/.bashrc`，导致 bashrc 中的环境变量覆盖了 `agent/.env` 中的值。

## 修复方案

### 临时方案

直接用 python 启动，绕过 bashrc：
```bash
python3.11 -m api_server --port 8899
```

### 永久方案

修改 `~/.bashrc` 中的 MiniMax 配置为正确值：
```bash
export MINIMAX_API_KEY="sk-F7HNrgA2Iaa3F0cxr4o7zmx6pq5yIo4cMDyxojwzJ8bcb"
export MINIMAX_BASE_URL="https://maas.gd.chinamobile.com:36007/ai/uifm/open/v1"
```

## 关键经验

1. **环境变量优先级**：当通过 `vibe-trading serve` 启动时，bashrc 中的环境变量会被加载，可能覆盖项目 .env 中的值。

2. **进程环境变量检查**：使用 `cat /proc/<pid>/environ` 可以直接查看运行中进程的实际环境变量，是排查"配置没生效"问题的关键手段。

3. **API 测试先行**：在排查代码问题前，先用 curl 确认 API 本身是否正常工作，可以排除配置问题。
